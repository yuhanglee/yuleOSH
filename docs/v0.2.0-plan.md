# yuleOSH v0.2.0 Sprint 1 — AI Pipeline 诚实化

## 目标
让 yuleOSH 的 Pipeline 真正调用 AI/LLM，而不是只生成模板 Markdown。

## 当前问题（已确认）
`src/pipeline/run.py` 共 9 个步骤：
- ✅ spec-check（这个可以保留，是本地 validate）
- ❌ super-analysis（模板填充，无 AI）
- ❌ prd（模板填充，无 AI）
- ❌ internal-review（文件存在性检查而已）
- ❌ claude-arch（目录扫描+模板）
- ❌ claude-dev（git log+文件计数）
- ❌ claude-test（跑 pytest，这个保留）
- ❌ hermes-review（静态 JSON）
- ✅ final-report（汇总报告，保留）
- ❌ **Hermes 和 Claude 的步骤没有实际调用 LLM API**
- ❌ **没有把任务路由到真正的 Agent**

## 执行方案

### Phase 1：Pipeline 接入 LLM API
1. 添加 `src/llm/client.py` — LLM API 客户端（OpenAI 兼容，复用 DeepSeek API）
2. 修改 pipeline：
   - `super-analysis` → 调 LLM 做真实 S.U.P.E.R 分析
   - `claude-arch` → 调 LLM 做真实架构分析
   - `claude-dev` → 调 LLM 生成开发计划
   - `hermes-review` → 调 LLM 做真实代码审查
3. 保留 `spec-check` 和 `final-report` 不变

### Phase 2：Pipeline 路由到真 Agent
1. Claude 步骤 → 把上下文传给 Claude sub-agent 执行
2. Hermes 步骤 → 同上，传给 Hermes
3. 小明步骤 → 小明自己做
