## 三、交付清单

### Task 1: QEMU 版本锁定 + 运行时校验

**文件**: `src/cross/sil_runner.py`

| 变更 | 说明 |
|------|------|
| `MIN_QEMU_VERSION = (8, 2, 0)` | 最低版本常量 |
| `MAX_QEMU_VERSION = (8, 3, 0)` | 最高版本常量 |
| `parse_qemu_version(version_output)` | 解析 `--version` 输出的版本三元组 |
| `_check_qemu_version()` | 启动时校验版本，过低抛 `RuntimeError`，过高 `log.warning` |
| 现有 60 测试 | 自动注入 mock 版本，不依赖真实 QEMU |

**文件**: `Dockerfile.cross`

- 基础镜像升级到 `ubuntu:24.04`（原生 QEMU 8.2.x）
- 安装 `qemu-system-arm` + `qemu-system-riscv64`
- `apt-mark hold` 锁定版本
- 启动时 `_check_qemu_version()` 双重校验

### Task 2: CI L2 sil-tests stage

**文件**: `src/ci/run.py`

| 函数 | 说明 |
|------|------|
| `run_sil_tests(project_dir, ci)` | 扫描 `tests/fixtures/prebuilt/*.elf`，对每个执行 `sil_test()` |
| L2 集成 | 在 cross-compile 之后、integration-tests 之前调用 |
| 阻断逻辑 | 失败时 return False，pipeline 中止 |
| 结果输出 | `.osh/ci/sil-test-results.json` |
| 空白跳过 | 无 .elf 时温和跳过，不视为失败 |

### Task 3: Evidence 包集成 SIL 报告

**文件**: `src/evidence/pack.py`

| 方法 | 说明 |
|------|------|
| `sil_reports: list[dict]` | 新的实例属性 |
| `collect_sil_reports()` | 扫描 `.osh/ci/*sil*.json` 加载为报告数据 |
| `generate_evidence()` | 自动调用 `collect_sil_reports()` |
| `pack_compliance_zip()` | SIL 报告以 `sil-reports/` 前缀纳入 compliance pack |

### Task 4: 预编译测试固件供应链

**新增目录**: `tests/fixtures/`

```
tests/fixtures/
├── README.md          ← 编译说明
├── hello-arm/
│   ├── hello.c        ← 最小 ARM 裸机固件 (UART 输出)
│   ├── startup.c      ← 向量表 + Reset_Handler
│   └── Makefile       ← arm-none-eabi-gcc 编译脚本
└── prebuilt/
    └── .gitkeep       ← CI 识别占位
```

hello.c 核心逻辑:
```c
void uart_send(const char *s) {
    volatile unsigned int *uart = (unsigned int *)0x40001000;
    while (*s) { *uart = *s++; }
}
void _start(void) {
    uart_send("Hello from yuleOSH cross-compilation test!\n");
    uart_send("Architecture: ARM\n");
    uart_send("Test Complete\n");
    while (1) {}
}
```
