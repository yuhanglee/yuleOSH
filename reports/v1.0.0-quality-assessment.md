### 3.2 🔴 覆盖率测量失效问题 (B-01)

**问题描述**: `.coveragerc` 使用 `[paths] source = src/yuleosh` + `*/site-packages/yuleosh` 的 paths 配置，但 `pytest.ini` 使用 `--cov=yuleosh`。当 yuleosh 未以 pip install -e . 安装时，coverage 无法正确映射源文件路径。

**当前状态**: 运行 `pytest --cov --cov-fail-under=80` 实际测得 **~11%**（仅少数正确映射的模块），远低于 README badged 的 86%。

**原因分析**:
- `.coveragerc` `[run] source = yuleosh` 期望 `yuleosh` 作为已安装包出现
- 本地开发时 `src/yuleosh/` 未被识别为 `yuleosh` 包
- `pyproject.toml` 的 `[tool.coverage.run] source = ["yuleosh"]` 同样不适用于 `python -m pytest` 直接运行

**修复方案**: 
- 安装包到开发模式: `pip install -e .` 后运行 coverage
- 或修改 pytest.ini: `--cov=src/yuleosh` 使用文件路径而非包名
- 或统一到 `pyproject.toml` 配置

## 9. 阻塞性问题汇总

### 🚨 B-01: 覆盖率测量失效（.coveragerc 路径配置问题）

| 属性 | 内容 |
|:-----|:------|
| **文件** | `.coveragerc`, `pyproject.toml`, `pytest.ini` |
| **严重度** | 🔴 **P0 阻塞** |
| **描述** | `.coveragerc` 中使用 `[paths] source = src/yuleosh` + `*/site-packages/yuleosh` 的路径映射，但 `pytest.ini` 使用 `--cov=yuleosh` 包名。当未 `pip install -e .` 时覆盖率无法正确测量，实际测得 ~11%。 |
| **影响** | - README 86% badge 不可信<br>- CI 中的 `--cov-fail-under=80` 形同虚设<br>- 无法追踪覆盖率改进趋势<br>- Sprint 1 门禁（store_pg ≥85% 等）无法验证 |
| **修复方案** | 1. 立即: 将 `pytest.ini` 中的 `--cov=yuleosh` 改为 `--cov=src/yuleosh`<br>2. 或统一配置到 `pyproject.toml`，删除 `.coveragerc`<br>3. 修复后重新运行全量 coverage，建立真实基线 |

### 🚨 B-02: evidence/pack.py 严重超标

| 属性 | 内容 |
|:-----|:------|
| **文件** | `src/yuleosh/evidence/pack.py` |
