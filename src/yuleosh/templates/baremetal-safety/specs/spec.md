## 1. System Requirements

### SR-001: Safety Architecture (ASIL-B)
- The system SHALL implement dual-core lockstep architecture
- The system SHALL provide independent watchdog supervision
- The system SHALL support ECC-protected memory access
- The system SHALL implement runtime self-test for critical functions

### SR-002: Fault Detection
- The system SHALL detect CPU register faults via periodic self-test
- The system SHALL monitor stack pointer integrity
- The system SHALL detect memory corruption via CRC checksums

---

## 2. Software Requirements

### REQ-001: Watchdog Management
- The system SHALL refresh the external watchdog within specified window
- The system SHALL enter safe state on watchdog timeout
- The system SHALL log watchdog events to EEPROM

### REQ-002: ECC Memory
- The system SHALL initialize SRAM with ECC scrubbing on boot
- The system SHALL handle single-bit error correction
- The system SHALL trigger fault handler on multi-bit errors

---

## 3. Test Plan

### Scenario: Watchdog Timeout
- GIVEN the watchdog timer is armed
- WHEN the watchdog is not refreshed within the timeout period
- THEN the system SHALL enter safe state
- AND the fault flag SHALL be set in EEPROM
